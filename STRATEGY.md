## 호스트 자격 조건

- 타 모임 **3회 참석** 시 자동으로 호스트 자격 획득
- 오너 승인 불필요 — 플랫폼 문화를 먼저 경험한 사람만 호스트 가능

### 호스트 전환 흐름

```
가입 (guest)
    ↓ 타 모임 3회 참석
호스트 자격 획득 (자동)
    ↓ "내 모임 열기" 클릭 (Phase 2 개발 예정)
모임 개설 → 진행 → 수익
(참가자 4명 초과 시 플랫폼 과금 발생)
```

### 운영자 단계별 역할

```
지금 ~ 3개월: 직접 모임 운영 → 호스트 매뉴얼 완성
3개월 ~ 6개월: 호스트 베타 3~5명에게 위임
6개월 이후: 플랫폼 운영/관리만
```

## 서비스 확장 방향 — FT (Facilitated Team) 플랫폼

ntable의 본질은 **호스트가 진행하는 구조화된 그룹 경험 엔진**이에요.
질문 세트만 바꾸면 어떤 모임에도 적용 가능합니다.

### 확장 가능한 버티컬

| 버티컬 | 콘텐츠 | 타겟 | 단가 |
|--------|------|------|------|
| 소셜 미팅 (현재) | 연애 성향 탐구 | 직장인 B2C | 2~4만원/인 |
| 팀빌딩 | 업무 스타일 탐구 | 기업 B2B | 5~10만원/인 |
| 멤버십 트레이닝 | 가치관/역할 탐구 | 동호회/커뮤니티 | 3~5만원/인 |
| 신입 오리엔테이션 | 조직문화 탐구 | 기업 HR | 5~10만원/인 |
| 스터디그룹 | 학습 스타일 탐구 | 교육기관 | 2~3만원/인 |

### 포지셔닝 전략 — 피보팅 아닌 확장

```
ntable.kr        → 플랫폼 (호스트가 여는 그룹 경험)
남산엔테이블      → 소셜 미팅 버티컬 (현재)
ntable for Teams → 기업 팀빌딩 버티컬 (Phase 3)
```

## 핵심 전략 인사이트

### 왜 구독 모델이 가능한가

> "애인 생기면 떠나는 거 아니야?" — 이 질문에 대한 답

소셜 모임의 본질은 **연애 성공**이 아니라 **괜찮은 사람들과 연결되고 싶은 욕구**.
이 욕구는 애인 생겨도 사라지지 않음.

**핵심 포지셔닝: "연애 앱"이 아닌 "괜찮은 사람들이 모이는 공간"**

### 피봇 시그널

- AI/재테크/와인 모임 → 4:4 미팅 피봇 후 **반응 폭발**
- 피봇 후 나간 10명 < 새로 들어온 15명 → 타겟 정제 성공
- 고대/연대/카이스트 직장인: 학교 때 자연스럽던 만남이 직장 오면 없어짐 → **니즈 정확히 충족**

### 성장 변곡점

| 시점 | 조건 | 액션 |
|------|------|------|
| 단기 | 멤버방 100명 | 자체 커뮤니티 도입 |
| 중기 | 모임 5회, 멤버 50명 | 호스트 베타 모집 |
| 장기 | 멤버 100명, 매출 발생 | 투자자 미팅 |

### 초대제 전략

기술적 초대제보다 **"지인 추천 기반 운영 방침"**이 현실적.
공개 단톡방(진입 낮음) → 멤버방(검증된 사람만) 2단계 구조 유지.

### 불참 대응

전날 리마인드 메시지 + 멤버방 "급참 가능" 공지 + 5:4로 여유있게 모집 후 당일 성비 조정.
